世のプログラマは皆CLIに詳しくてzshやfishを細かくカスタムしており、何らかのCLIコマンドプログラムを書いたことがある(要出典)が、
この記事では打ちやすいコマンド、打ちにくいコマンドをいろいろ紹介していき __打ちやすいコマンド、打ちにくいコマンドとはどのようなものなのか__ という問題を考察していく。

## asdf

優勝を飾ったのはCLIツールのためのバージョンマネージャであるasdfだ。ホームポジションで左手で小指から人差し指までシャララーンと打つだけと非常にかんたん。

ただしサブコマンドに打ちづらいものが多く、`asdf plugin list all` とか `asdf install nodejs 14.9.0` とか挙句 `asdf install java adoptopenjdk-14.0.2+12` とか打たなければならないのがつらい。
シェル補完を入れると adoptopenjdk～とかにも抜群にすぐれた補完が効くのでぜひとも入れておきたい。

## brew

macOS/Linux向けパッケージマネージャ Homebrew のコマンド。Bキーを左人差し指で打つとRがクッソ打ちづらくなるので右で打つとよい。

## node

JavaScriptエンジンNode.jsのコマンド。deで左中指が連続するように見えるがDを人差し指で打つことによって回避できる。

## npm

Node.jsのパッケージマネージャ。キーの移動が多く大変そうにみえるが、Pを薬指で打つために手が右に移動するため自然に人差し指がMの位置に移り、そこまで打ちづらくはない。
少なくともnpnとかpnpとかよりはマシだろう。

npmの特筆すべき点はサブコマンドの省略が可能なところで、npm installはnpm iと省略できる。賛否あると思うが個人的にはそこそこ推せる。

## emacs

なにこのコマンド打ちづら過ぎて死ぬかと思った。emaはいいんだがacsと左手外側の動かしづらい指が変な順番で出てくるのが地獄すぎる。
_Escape-Meta-Alt-Control-Shift_ と揶揄されるだけあってこの程度スラスラ打てるようにならねばemcasは使えないということか。

なんやこのクソvimmerと思った人は10秒待ってほしい。筆者はVSCode使いだ。

## code

Visual Studio Code をCLIから起動するときのコマンドだ。nodeと類似しているように普通に打ちやすい。少なくともemacsよりは。
