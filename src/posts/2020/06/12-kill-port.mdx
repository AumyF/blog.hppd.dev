title: webpack-dev-serverを強制終了してポートが専有されっぱになってるときの対処
date: 2020-06-12
tags: 
  - Linux
---

Hexoとかでもあるように、Gatsbyにも開発サーバーを建ててローカルでサイトをプレビューできる機能があります。
で、dev server が重くなってるのでってCtrl+Cを連打するとプロセスが強制的に殺されてプロセスが一部生き残ってしまいます。
そんでその生き残ってるプロセスは往々にしてポートを LISTEN したまま仁王立ちで死んでいるため、再度 server を立ち上げようとすると
`Error: listen EADDRINUSE: address already in use :::8000` で落ちてしまうんですね。そんなときのための備忘録です。
